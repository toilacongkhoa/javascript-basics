# Một số khái niệm quan trọng trong JavaScript: Callback, Vòng lặp và Đối tượng

Tài liệu này giải thích chi tiết về một số khái niệm cơ bản nhưng rất quan trọng trong quá trình làm việc với JavaScript: Callback function, cách lặp qua mảng/đối tượng và cách kiểm tra thuộc tính của đối tượng.

## 1. Callback Function (Hàm gọi lại)

**Định nghĩa:** Callback là một hàm được truyền vào một hàm khác dưới dạng một đối số (argument) và sau đó được gọi lại (thực thi) bên trong hàm bên ngoài đó để hoàn thành một hành động hoặc một nghiệp vụ nào đó.

**Tại sao cần dùng callback?**
- **Xử lý bất đồng bộ (Asynchronous):** Trong JavaScript (đặc biệt là với các tác vụ như gọi API, đọc file, `setTimeout`), code sẽ không chạy tuần tự chờ từ trên xuống dưới. Callback giúp ta đảm bảo một hàm chỉ được chạy *sau khi* một tác vụ khác đã hoàn thành.
- **Tái sử dụng code:** Rất nhiều phương thức có sẵn của JavaScript (như `Array.prototype.map`, `filter`, `reduce`, đính kèm sự kiện `addEventListener`) hoạt động dựa trên việc nhận vào một callback để xử lý trên từng phần tử hoặc khi xảy ra sự kiện.

**Ví dụ:**
```javascript
// Hàm chính nhận vào một tham số và một callback
function processData(data, callback) {
    console.log("Đang xử lý dữ liệu: " + data);
    // Gọi lại hàm callback sau khi quá trình xử lý hoàn tất
    callback(data);
}

// Hàm callback
function showResult(result) {
    console.log("Kết quả đã xử lý xong: " + result);
}

// Truyền hàm showResult như một argument vào hàm processData
processData("Dữ liệu tuyệt mật", showResult);
```

## 2. Vòng lặp và sự thay đổi độ dài (length) của mảng

Trong Javascript, mảng (Array) thực chất là một đối tượng đặc biệt, và thuộc tính `length` có thể thay đổi động trong quá trình lặp. 

**Vấn đề với vòng lặp `for` truyền thống:**
Nếu trong quá trình lặp bằng `for (let i = 0; i < arr.length; i++)`, bạn vô tình thêm hoặc xóa phần tử khiến `arr.length` thay đổi liên tục, kết quả có thể dẫn đến việc lặp vô hạn hoặc bị bỏ lọt các phần tử.

**Các giải pháp lặp qua mảng an toàn và hiện đại hơn:**
- **`for...of` (Khuyên dùng cho mảng):** Trực quan và đơn giản nhất để lặp qua **giá trị** của các phần tử trong một mảng. Nó tự động xử lý các phần tử được lặp qua mà không phụ thuộc trực tiếp vào biến bộ đếm `i` hay `length` ở mỗi vòng lặp.
  ```javascript
  const arr = ['A', 'B', 'C'];
  for (const item of arr) {
      console.log(item); // Lần lượt in ra A, B, C
  }
  ```
- **Lưu ý về `for...in` với mảng:** Mặc dù mảng là object và *có thể* dùng `for...in` để lặp qua khóa (chính là chỉ số index `0`, `1`...), nhưng **không khuyến khích** sử dụng cách này. `for...in` lặp qua tất cả các thuộc tính dạng chuỗi ('0', '1',...) và có thể lặp qua cả những thuộc tính custom được thêm vào mảng hoặc trong nguyên mẫu (prototype) của nó, thứ tự lặp cũng không được đảm bảo chắc chắn trình tự.

## 3. Lặp qua Object bằng `for...in`

Vòng lặp `for...in` được thiết kế đặc biệt để lặp qua tất cả các **thuộc tính đếm được (enumerable properties)** cục bộ của một đối tượng.

**Lưu ý quan trọng:** 
Trong JavaScript, `for...in` sẽ quét qua **tất cả các keys (khóa)** của đối tượng. Điều này có nghĩa là nó lặp qua:
1. Các **thuộc tính (properties)** tùy chỉnh chứa giá trị.
2. Các **phương thức (methods)** - xét cho cùng phương thức là các thuộc tính mang giá trị là một hàm (function).
3. **Đặc biệt (Prototype Chain):** Các khóa có trong cả **chuỗi Prototype (Nguyên mẫu)** mà đối tượng kế thừa, miễn là chúng được đặt ở chế độ *enumerable* (có khả năng đếm được).

**Ví dụ lặp qua cả thuộc tính và phương thức từ prototype:**
```javascript
function Person(name) {
    this.name = name; // Thuộc tính tự thân
}
// Thêm một phương thức vào Prototype của Person
Person.prototype.sayHello = function() {
    console.log("Hello!");
};

const user = new Person("Nguyễn Văn A");

// Tiến hành lặp qua đối tượng user
for (const key in user) {
    console.log(key); 
    // Console sẽ in ra lần lượt: 
    // "name" (thuộc tính tự thân) 
    // "sayHello" (phương thức bị kế thừa từ lớp Prototype)
}
```

## 4. Kiểm tra đối tượng với `hasOwnProperty()`

Do nhược điểm của `for...in` là lặp qua cả các thuộc tính kế thừa từ Prototype như đã ví dụ ở trên, lập trình viên cần một công cụ để kiểm tra xem một thuộc tính có thực sự *thuộc quyền sở hữu độc lập* của riêng đối tượng đó hay không. Đó là lúc `hasOwnProperty()` phát huy vai trò quan trọng.

**Định nghĩa:** `hasOwnProperty(key)` là một phương thức có sẵn trong các Object, nó lấy một chuỗi `key` làm đối số và trả về `true` hoặc `false`.
- Trả về **`true`**: Khi thuộc tính/phương thức đó thực sự nằm và được khởi tạo trực tiếp trên bản thân đối tượng hiện tại.
- Trả về **`false`**: Kiếm tra ra thuộc tính không tồn tại, HOẶC thuộc tính đó được kế thừa thông qua chuỗi Prototype cha.

**Cách áp dụng Best Practice đối với vòng lặp Object:**
Đây là mẫu lệnh kinh điển (Best Practice) khi muốn truy xuất và xử lý dữ liệu qua object một cách cô lập:

```javascript
const car = { brand: 'Toyota', color: 'Black' };

// Chẳng may có một thư viện nào đó chỉnh sửa Object nguyên mẫu
Object.prototype.wheels = 4;

for (const key in car) {
    // Luôn bọc điều kiện hasOwnProperty để loại bỏ các thuộc tính rác từ Prototype
    if (car.hasOwnProperty(key)) {
        console.log(`Thuộc tính tự sở hữu: ${key} = ${car[key]}`);
        // Chạy đoạn mã này thì sẽ chỉ in ra `brand` và `color`. "wheels" sẽ bị bỏ qua.
    }
}
```
